# Rules and regulations (and features) while writing mdx

> MDX has (almost) all features of normal markdown

## Headers

```
# Header 1
## Header 2
...
###### Header 6
```

- Headers when clicked copy the URL to directly focus onto them
- Only the first header of the file should be a Header-1 (single hash)

## Text formatting

1. **Bold** `**Bold**`
2. _italics_ `_italics_`
3. ~striked~ `~striked~`
4. `inline code` \`inline code\`
5. Code blocks

````
```<language>
<code>
```


```py
<code>
for i in range(10):
    print("hello")
```
````

6. Tables - normally

## Embedding images

- Save images in `/public/images` folder and reference them here using `/images/<image name>`
- DO NOT directly use the `<img/>` tag to embed images since they wont get styled automatically

## Embedding React components

Since we are using MDX we can import and export any component to and from the .mdx files


There are two ways to go about when including custom components

1. Direct imports
```tsx
import MyComponent from "@/components/MyComponent"

<MyComponent/>
```

Example
```tsx
import Sample from "@/components/Sample"
<Sample text={"this is it being directly imported into the mdx file"/> <Sample text={"this is it being directly imported into the mdx file"/>
```

import Sample from "@/components/Sample"

<Sample text={"this is it being directly imported into the mdx file"}/>

2. We define it globally by adding it to the global `Elements` map object
    - Goto _/src/content/registry.ts_
    - Import and add the element component to the object passed to `generateElementsFrom` function, and then directly reference that component here

```tsx
import {generateElementsFrom} from "@d1vij/jassm";
import Sample from "@/components/Sample";

export const Elements = generateElementsFrom({
    Sample, // this will automatically be injecting in all mdx files
}, true);
```

> the build step **WONT FAIL** if the compiler is unable to resolve the component at build time, instead it would fail at runtime

## Blog Metadata

Each blog **must** export a metadata object. The schema for which is present at `/src/schemas/BlogExportSchema.ts`

Example 

```ts
export const meta = {
  author: "Divij Verma",
  created_at: "26-02-2026",
  modified_at: "27-02-2026",
  tags: ["tutorial", "style guide", "mdx"],
};
```

If the exported meta is not upto the schema, runtime error will be raised
